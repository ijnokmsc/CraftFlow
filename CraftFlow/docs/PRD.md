# CraftFlow — FF14 生产辅助插件 PRD

## 项目信息

| 字段 | 值 |
|---|---|
| Language | 中文 |
| Programming Language | C# / .NET 8 / Dalamud API 15 |
| UI Framework | ImGui（即时模式渲染） |
| Project Name | craftflow |
| 版本 | v2.0（重构版） |

### 原始需求复述

1. **装备选择流程重构**：当前按 CraftType（制作职业）筛选装备不合理，应改为按战斗/采集/生产角色分组，再选职业，再选装备槽位，支持一键添加整套/主副手/首饰
2. **材料清单剔除水晶/晶簇**：水晶和晶簇为广泛获取的基础材料，在 BOM 展开结果中应默认过滤
3. **GBR IPC 初始化时序问题**：插件构造函数中调用 `Version()` 验证 GBR 可用性，但 GBR 此时可能未加载完成，导致 `IsAvailable=false`
4. **收藏队列和推荐队列**：增加收藏功能（玩家自行添加）和推荐队列（内置常用套装，如 770HQ 坦克套、750HQ 生产套、750HQ 采集套）

## 2. 用户故事

### US1: 按角色分组选择装备（对应需求 1）

> 作为一个 **FF14 玩家**，我想要 **先选择角色分组（防护/治疗/DPS 等），再选择具体职业，再按槽位浏览装备**，以便 **快速找到我角色需要的整套装备，而不是在制作职业的下拉框中翻找**。

### US2: 一键添加整套装备（对应需求 1）

> 作为一个 **FF14 玩家**，我想要 **一键添加整套/主副手/防具/首饰到制作列表**，以便 **不用逐个勾选 12 个装备部件**。

### US3: 材料清单剔除水晶晶簇（对应需求 2）

> 作为一个 **使用 CraftFlow 规划制作的玩家**，我想要 **材料清单默认不显示水晶和晶簇**，以便 **聚焦于需要规划采集和制作的材料，而不是被大量基础材料淹没**。

### US4: GBR 可用性检测准确（对应需求 3）

> 作为一个 **同时安装了 GBR 的玩家**，我想要 **CraftFlow 正确识别 GBR 已安装**，以便 **能直接使用推送采集功能，而不需要重载插件**。

### US5: 收藏队列和推荐队列（对应需求 4）

> 作为一个 **经常制作固定套装的玩家**，我想要 **收藏常用的装备组合，并从推荐队列中快速选择常用套装**（如 770HQ 坦克套、750HQ 生产套），以便 **不用每次都重新选择**。

## 附录 C：GBR IPC 延迟检测方案

**问题根因**：`Plugin.cs` 构造函数中 `try { _gbrIpc.Subscribe(); } catch { }` 立即调用 `Version()`，此时 GBR 可能尚未完成 IPC 注册。

**修复方案**：延迟到 `Framework.Update` 事件中首次检测，带指数退避重试：

```csharp
// Plugin.cs 构造函数中
_gbrIpc = new GbrIpcClient(PluginInterface, Log);

// 移除立即订阅
// try { _gbrIpc.Subscribe(); } catch { }

// 改用延迟订阅
var retryCount = 0;
var maxRetries = 10;
IFramework? framework = null; // 需注入

framework!.Update += OnFrameworkUpdate;

void OnFrameworkUpdate(IFramework fw)
{
    if (_gbrIpc.IsAvailable || retryCount >= maxRetries)
```
